Pre Deployment Instructions
Deployment Instructions
  • 1. Download artifacts from artifactory
    {{artifactName}}
    {{artifactVersion}}
  • 2. Deploy SPA application
  • 3. Get the Bearer Token for this environment using right client id and secret
  • 4. Go to the upload Zip collection in Postman, if you do not have the collection use the below to import into postman:
  • Use the following BaseURL’s
    baseurl CPP (PP-SHELL-STG) – https://api.us1.pp.covisint.com
    baseurl for PROD(SHELL-PROD) - https://api.us1.covisint.com 
  • curl -X POST \
  <<baseurl>>/proxyadmin/v2/upload \
  -H 'Accept: multipart/form-data' \
  -H 'Authorization: Bearer <<token>>' \
  -H 'Cache-Control: no-cache' \
  -H 'Postman-Token: 5f3bd857-4f50-4d73-9676-09ba3dc91202' \
  -H 'content-type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW' \
  -H 'origin-url: <<originurl>>' \
  -F zipfile=<<zip file>>
  • 5.Replace the Bearer token and originurl to sipsapp.stg.sips.shell.com
  • 6. Click on the Body tab in the postman and select “form-data” choice button
  • 7. In key column, type “zipfile” and select File from dropdown.
  • 8. Click on Choose Files button and select the file downloaded from the artifactory location mentioned above
  • 9. Verify if all the steps have been executed correctly and Bearer & originurl value has been put correctly
  • 10. Click on Send.
  • 11. In response a Job_Id will be sent, which needs to be put into the change control CRT. Also Update the ticket with artifact file name and SVN location from where you downloaded.
  • 12. You can verify job status using 
    curl -X GET \
  'https://api.us1.pp.covisint.com/proxyadmin/v2/status?jobid= {{jobId}}'\
  -H 'Authorization: Bearer token' \
  -H 'Cache-Control: no-cache' \
  -H 'Postman-Token: fb48a93e-1626-f855-9a12-7af63134d142' \
  -H 'origin-url: 'originurl''
